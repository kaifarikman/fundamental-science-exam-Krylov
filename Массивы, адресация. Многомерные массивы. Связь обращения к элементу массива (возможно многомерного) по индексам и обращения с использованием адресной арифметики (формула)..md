### **Массивы и адресация**

  

**Массив** — регулярный структурный тип с **индексированным методом доступа**:

- все компоненты **одного типа**;
    
- индексная структура — **декартово произведение отрезков перечислимых типов** (то есть можно иметь один индекс, два индекса, …). 
    

  

Из этого сразу следует, что **многомерный массив** — это массив, у которого индекс — **набор индексов** (пара, тройка, …) из этого декартова произведения. 

---

### **Связь индексов и адресной арифметики (формулы)**



#### **1) Одномерный массив**

Если a — массив элементов типа T, базовый адрес base = &a[0], размер элемента s = sizeof(T), то:

$\text{addr}(a[i]) = base + i\cdot s$

В C это же записывается через указатели:

$a[i] \equiv *(a+i)$

  

#### **2) Двумерный массив (C, построчное хранение — row-major)**

Пусть a имеет размер N × M (N строк, M столбцов), тогда:

$\text{addr}(a[i][j]) = base + (i\cdot M + j)\cdot s$

и эквивалентная форма через указатели:

$a[i][j] \equiv *(*(a+i)+j)$

  

#### **3) k-мерный массив (общая идея)**

Если размеры по измерениям: $n_2$, $n_3$, $\dots$, $n_k$, то смещение считается “внутрь”:

$\text{offset} = (((i_1\cdot n_2 + i_2)\cdot n_3 + i_3)\cdots \cdot n_k + i_k)$

$\text{addr} = base + \text{offset}\cdot s$
