## **struct**

##  **— структура**

  

**struct** объединяет несколько полей **разных типов** в один объект.

  

Применение:

- описать “запись” с разными характеристиками (человек, товар, точка, студент и т.д.).
    

  

Пример:

```c
struct Point { int x; int y; };
struct Point p = {1, 2};
```

---

## **union**

##  **— объединение**

  

**union** — как struct, но **все поля занимают одну и ту же память**.

То есть одновременно “хранится” корректно **только одно** из полей (то, которое ты последним записал).

  

Применение:

- экономия памяти,
    
- “одно из вариантов” (variant), когда значение может быть разного типа.
    

  

Пример:

```c
union U { int i; float f; };
union U u;
u.i = 10;   // сейчас валиден i
u.f = 1.5f; // теперь валиден f, i уже “сломается”
```

## **union**

- struct: память = **сумма** полей (плюс выравнивание), все поля живут одновременно.
    
- union: память = **максимум** из размеров полей, поля делят одно место.

## **typedef**

  

**typedef** создаёт **псевдоним типа** (удобное имя).

  

Применение:

- упростить длинные типы,
    
- красиво писать struct.
    

  

Примеры:

```c
typedef unsigned long ulong;

typedef struct {
    int x, y;
} Point;

Point p;
```
